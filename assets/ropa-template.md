# Records of Processing Activities (ROPA) — template

> One table row per **processing activity** (Art. 30). This is the backbone document: most SME engagements
> start here because it forces the whole data map into view. Keep it current. See
> `references/records-and-accountability.md`.

## Header (organisation level)
- **Controller:** name, address, contacts.
- **DPO / representative:** name, contacts (or "not appointed", with the reasoning).
- **Last updated:** __________  **Owner:** __________

## Cross-checks once the ROPA is populated
- Every activity has a **valid lawful basis** (and an Art. 9 gateway if sensitive).
- Every processor has a **DPA**; every sub-processor is listed.
- Every transfer outside the EEA has a **tool + TIA**.
- Retention periods are defined (not "forever").
- High-risk activities have a **DPIA** (see `dpia-template.md`).
- A **privacy notice** covers each purpose.

> This ROPA is a preparation document, not a certification of compliance. Flag ⚠ items for verification
> against the current official sources and ⛔ items for the DPO/lawyer.
